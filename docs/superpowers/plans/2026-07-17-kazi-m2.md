# kazi M2 — Catalog & Lifecycle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** `kazi try postgres` (offline, zero residue on Ctrl-C), ad-hoc `kazi run <image>` stacks with routing, `kazi adopt` for hand-run containers, `kazi gc`, `kazi eject`, and a template catalog (embedded starters + import + `template new --from-image` scaffolder).

**Architecture:** New package `internal/template` (catalog, values, scaffolder — pure logic + embedded starters). The engine gains a per-source strategy (`compose`/`template` | `image` | `containers`) behind `resolve`, cashing in the M0 forward-compat promise: `ls/status/urls/ps` render all sources unchanged; only lifecycle verbs dispatch per source. All container operations remain subprocess-only via the existing `runtime.Runtime` (no interface changes — `Cmd` covers run/inspect/network/start/stop).

**Tech Stack:** existing deps only (cobra, yaml.v3) + `go:embed` for starters. Git import shells out to `git`.

**Spec:** `docs/superpowers/specs/2026-07-17-kazi-m2-design.md` — read before any task. M0/M1 rules still bind (`CLAUDE.md`).

## Global Constraints

- All logic in `internal/engine` + `internal/template` (+ existing packages); cobra handlers stay thin. Runtime interaction subprocess-only.
- Schema stays `kazi.dev/v1alpha1`, all M2 fields additive: `spec.source` arms `image:`/`template:`/`containers:` become real (still a union — exactly one arm), `spec.ephemeral`, `spec.values`, `spec.volumes`, `metadata.createdAt` (RFC3339); config `spec.cleanup.ephemeralTTL` default `"24h"`.
- **Ephemerality lives in the manifest** (`spec.ephemeral`); the `kazi.ephemeral` container label is written at creation only, as a crash hint for gc. `--keep`/`kazi keep` edit the manifest and NEVER recreate containers.
- **Adopted containers are never recreated or relabeled** (labels immutable — hard constraint). Membership is `spec.source.containers` only.
- **gc invariant:** every ephemeral artifact carries a manifest or the crash-hint label; partial teardown always leaves one of the two.
- kazi never overwrites an existing template dir; collisions are errors suggesting a name.
- Templates use plain compose with `${VAR:-default}` interpolation only — no templating language.
- Agent contract everywhere: `--json`, exit codes 0/1/2/3/4, `--yes` on prompts, idempotent verbs. Stack names are DNS labels.
- Embedded starters (exact set): postgres, redis, mysql, mongo, mailpit, minio.
- Subagent models: sonnet default, opus for T4 (strategy refactor) and the final review — never haiku.
- Commit style `feat:/fix:/test:/docs:/chore:` + trailer `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.

### Task 4 (opus, solo): Engine source strategies + grouping/routing for image & containers

**Files:** Modify `internal/engine/engine.go` (resolve/target), `internal/engine/view.go` (snapshot grouping), `internal/engine/lifecycle.go` (verb dispatch); create `internal/engine/strategy.go`, `internal/engine/strategy_test.go`.

**Design (implement exactly):**

1. **target** gains `srcKind string` ("compose"|"image"|"template"|"containers"), `image string`, `containers []string`, `manifest *store.Manifest` (loaded once in resolve; nil for discovered). `resolve` sets srcKind from `m.Spec.Source.Kind()` (registered) or "compose" (discovered). Template stacks: `dir` = materialized template dir (`template.Materialize`), `composeFile` = `<dir>/compose.yml`.
2. **Env interpolation for template stacks:** effective values = template `LoadValues` defaults ← manifest `Spec.Values`; `FlattenEnv` appended to `cmd.Env` (`append(os.Environ(), env...)`) for every compose command of that stack. Thread through a small helper `composeCmdFor(t target, args ...string) *exec.Cmd` used by all compose-strategy verbs.
3. **strategy dispatch** in `strategy.go`:

```go
type strategy interface {
	up(ctx context.Context, e *Engine, t target) error
	down(ctx context.Context, e *Engine, t target, extraArgs ...string) error
	restart(ctx context.Context, e *Engine, t target) error
	logs(ctx context.Context, e *Engine, t target, service string, follow bool, tail string) error
}
func strategyFor(kind string) strategy // compose & template → composeStrategy; image → imageStrategy; containers → containersStrategy
```
Existing `Up/Down/Restart/Logs` bodies move into `composeStrategy`; public verbs become `resolve → strategyFor(t.srcKind).<verb> → syncProxy` (unchanged warning semantics).

4. **imageStrategy** (container name `kazi-<stack>`, single service named `<stack>`):
   - `up`: if container exists (ps by name) → `rt.Cmd("start", cname)`. Else create: classify from `spec` (see 6) — if routable, `proxy.EnsureNetwork` first — then one `rt.Cmd("run", "-d", "--name", cname, "--label", "kazi.managed=true", "--label", "kazi.stack="+name, [--label kazi.ephemeral=true if manifest.Spec.Ephemeral], [--network kazi --network-alias <stack>.<stack> if routable], each `-e K=V` from Spec.Values (upper-cased keys), each `-p` from Spec.Expose (pinned: `hostPort:containerPort`), each `-v` from Spec.Volumes, imageRef)`. Stream via compose.Run.
   - `down`: `rt.Cmd("stop", cname)` (stop, not rm — manifest owns identity).
   - `restart`: `rt.Cmd("restart", cname)`; `logs`: `rt.Cmd("logs", [-f], [--tail N], cname)`.
5. **containersStrategy** (adopted): `up` = `start <names...>`, `down` = `stop <names...>`, `restart` = `restart <names...>`; `logs` follows the FIRST container when service=="" (or the named one when service matches a container name). Never create/rm.
6. **Classification without compose:** add `func classifyPorts(ports []int, cfg store.Config) (httpPort int, ok bool)` reusing `proxy.Classify` with a synthetic `compose.ServiceInfo`. Image stacks: ports from `rt.Cmd("image", "inspect", "--format", "{{json .Config.ExposedPorts}}", ref)` — parse `{"5432/tcp":{}}` keys (helper `parseExposedPorts([]byte) []int` — unit test it). Containers stacks: ports parsed from the container's ps `Ports` string (`parsePsPorts(s string) []int` — handles `"0.0.0.0:8080->80/tcp, 6379/tcp"` → container-side ports; unit test).
7. **snapshot grouping** (view.go): containers with no compose-project label but a `kazi.stack` label group under that stack (Service = the stack name for image stacks). Registered manifests with source image/containers claim their containers by `kazi.stack` label (image) or by container NAME membership (adopted — names from `spec.source.containers`; these containers keep kind registered even without kazi labels). Everything else falls through unchanged.
8. **desiredRoutes** (lifecycle.go): for non-compose/template stacks, skip `compose config`; build the route from strategy data — image: alias `<stack>.<stack>`, hostname `<stack>.localhost`, port from classification (skip if none); containers: one route per adopted container that classifies HTTP — alias `<container>.<stack>`, hostname: single adopted HTTP container → `<stack>.localhost`, otherwise `<container>.<stack>.localhost`. Template stacks route exactly like compose (they ARE compose).

**Tests (strategy_test.go, Fake-based, reuse helpers):** `TestSourceKindDispatch` (image manifest: Up on fresh state records `run -d --name kazi-app` with both kazi labels; Up again with the container present records only `start`); `TestImageUpRoutable` (image inspect scripted via CmdOut → routable → `network create` + `--network kazi` + alias args present; Caddyfile contains `app.localhost`); `TestImageDownStops` (down records `stop kazi-app`, never `rm`); `TestContainersLifecycle` (adopted two names: up records `start a b`; down `stop a b`; containers never recreated — no `run`/`rm` calls ever); `TestParseExposedPorts` + `TestParsePsPorts` table tests; `TestTemplateStackEnvInterpolation` (template manifest + values → the recorded compose call's cmd.Env contains `POSTGRES_DB=custom`); `TestSnapshotGroupsKaziStackLabel` (container with only kazi labels + manifest → registered, not unmanaged). ALL existing M0/M1 engine tests must still pass unchanged (the refactor is behavior-preserving for compose stacks).

**Commit:** `feat: per-source lifecycle strategies — image and adopted-container stacks`

type TryOpts struct {

// Try: materialize template → pick a free DNS name (template name, else

### Task 7: gc

**Files:** Create `internal/engine/gc.go`, `internal/engine/gc_test.go`.

**Produces:**

```go
type GcItem struct {
	Kind   string `json:"kind"`   // "stack" | "container" | "allocation"
	Name   string `json:"name"`
	Reason string `json:"reason"` // "stopped ephemeral" | "ephemeral TTL expired (24h)" | "orphaned ephemeral container (crash hint)" | "allocation for missing stack"
}

// GcPlan selects, in spec order:
//  1. ephemeral manifests whose stack is fully stopped, OR older than
//     config cleanup TTL (parse via time.ParseDuration; createdAt absent ⇒
//     treat as expired);
//  2. containers wearing label kazi.ephemeral=true whose kazi.stack has NO
//     manifest;
//  3. port allocations whose stack has neither manifest nor containers.
func (e *Engine) GcPlan(ctx context.Context) ([]GcItem, error)

// GcRun executes a plan: stacks → Teardown (from T5); containers →
// `rt.Cmd rm -f <name>` then best-effort `volume`/image pruning is NOT
// attempted for orphans (only labeled containers are provably kazi's);
// allocations → FreeStack. Continues past per-item errors, returns the
// reclaimed list + joined error.
func (e *Engine) GcRun(ctx context.Context, items []GcItem) ([]GcItem, error)

// GcDebris is the status nudge: count of reclaimable items.
func (e *Engine) GcDebris(ctx context.Context) int
```
